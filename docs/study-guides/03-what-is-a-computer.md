# Study Guide 3: What is a Computer?

This topic is taught in lecture only, with no hands-on activity or
assignment. It lays the conceptual foundation for everything that follows:
what a computer actually is at the hardware level (CPU, memory, storage,
the fetch-decode-execute cycle) before the course moves into writing and
compiling code that runs on one.

## Know before you start

- Nothing beyond row 2's environment setup is required to follow the
  lecture material. [assumed: row 2 -- Environment Setup]

## Taught here

Concept: computer architecture fundamentals (lecture only, no artifact to
mine for verified content)
- Know that a computer executes instructions by repeatedly fetching an
  instruction from memory, decoding it, and executing it (the
  fetch-decode-execute cycle).
- Know that memory (RAM) and long-term storage (disk/SSD) are distinct:
  RAM is fast and volatile (cleared on power-off), storage is slower and
  persistent.
- Know that the CPU (central processing unit) is the component that
  executes instructions, and that a program is a sequence of instructions
  stored in memory.
