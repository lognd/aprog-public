# Activity: C-String Whodunit

## What you will practice

Recognizing classic C-style string bugs: buffer overflows, pointer comparisons
used instead of strcmp, writing to string literals, sizeof vs strlen confusion,
and strncpy's silent null-terminator omission.

## How it works

Seven buggy or tricky code situations are described one at a time.  For each
one, diagnose what is wrong (or what happens) and type your answer.  Correct
diagnoses unlock an explanation of the real failure mode.

No compilation needed -- read, reason, and answer.

## Running the activity

    python3 launch.py
